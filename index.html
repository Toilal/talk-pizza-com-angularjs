Avant, on avait JQuery

Ça marchait très bien !

Mais JQuery ...

  • N'aide pas à structurer le code
  • Ne fait que de la manipulation DOM et de l'AJAX
  • Provoque souvent du spaghetti code !

AngularJS, c'est l'anti JQuery

Framework complet

  • Ne dépends pas de JQuery.
  • Permet d'enrichir HTML (directives)
  • Conçu pour consommer une API REST
  • Intègre une gestion de modules
  • Sépare les responsabilités (MVC)
  • Utilise l'injection de dépendance
  • Est facilement testable

Les controllers

Le controller contient les données du modèle et publie les fonctions utiles pour la vue.

html
<div ng-controller="ResetCtrl as ctrl">
    <input ng-model="value">
    <button ng-click="ctrl.reset()">Reset</button>
    <span class="small">{{ctrl.value}}</span>

javascript
.controller('ResetCtrl', function() {
    this.value = 'lorem ipsum'; // Mise à jour automatiquement par binding

    this.reset = function() {
        this.value = '';
    }
});

Les directives

Elles encapsulent un modèle, un controlleur, et une vue (template).

javascript (directive)
.directive('resetInput', function() {
    // Corresponds à l'élément <reset-input></reset-input>
    return {
        restrict: 'E', // tag uniquement
        scope: { // => modèle
            label: '@', // attribut
            value: '=', // expression
            resetValue: '=?' // expression (optionel)
        },
        templateUrl: 'examples/reset.html', // => vue
        controller: 'ResetDirectiveCtrl', // => controller
        controllerAs: 'ctrl' // Nom du controleur au sein du template
    };
});

.controller('AngularDirectivesCtrl', function() {
    this.components = [
        {label: '#1', value: 'Premier', resetValue: 'Le numero 1'},
        {label: '#2', value: 'Deuxième', resetValue: 'Le numero 2'},
        {label: '#3', value: 'Troisième', resetValue: 'Le numero 3', hidden: true},
        {label: '#4', value: 'Quatrième', resetValue: 'Le numero 4'}
    ];

    this.handleClick = function(component) {
        // On sélectionne le composant s'il n'est pas déja selectionné, sinon le le déselectionne.
        this.selectedComponent = component != this.selectedComponent ? component : null;
    }
});

Services, Factories et Providers

Composants utilisables via l'injection de dépendance

  • Ces concepts permettent de palier à l'absence de classes en JavaScript
  • Equivalent Spring/Jersey en Java
  • Structure fortemment le code en séparant les responsabilités
  • Permet le remplacement des implémentations par des mocks pour les tests (angular-mocks)

.service('GithubApiService', function($http) {
    // $http est un service angularJS qui permet de réaliser des requetes HTTP.
    var base = "https://api.github.com/";

    var access_token = "1ab41bc0cf53c79f3240e07bfe1269705e013246";
    var headers = {
        'Accept': 'application/vnd.github.v3+json',
        'Authorization': 'token ' + access_token
    };

    this.user = function() {
        return $http({
            method: 'GET',
            headers: headers,
            url: base + 'user'
        })
    };

    this.userRepos = function(params) {
        return $http({
            method: 'GET',
            headers: headers,
            url: base + 'user/repos',
            params: params
        })
    };

    this.loadLink = function(url, params) {
        return $http({
            method: 'GET',
            headers: headers,
            url: url,
            params: params
        })
    }
})

javascript (controller)
.controller('GithubReposCtrl', function($scope, $q, GithubApiService) {
    // On récupère le service GithubApiService par injection de dépendance.
    var self = this;

    this.refreshing = false;

    this.refresh = function() {
        self.refreshing = true;

        var p1 = GithubApiService.user().then(function(response) {
            self.user = response.data;
            rateUpdater(response); // Rafraichir les compteurs
        }).catch(errorHandler);

        var p2 = GithubApiService.userRepos($scope.reposParams).then(function(response) {
            self.repos = response.data;
            loadNextRepos(response); // Charger l'ensemble des pages
            rateUpdater(response); // Rafraichir les compteurs
        }).catch(errorHandler);

        return $q.all([p1, p2]).finally(function() {
            self.refreshing = false;
        });
    };

    this.refresh(); // On rafraichit à l'initialisation du controlleur
})

AngularJS n'est pas parfait

  • Le binding cause des problèmes de performance (Dirty Checking, Digest Loop)
  • Il faut limiter la quantité de données reliées à la vue
  • On doit parfois "éviter le framework".
  • Ne supporte pas ES2015
  • Pas de Virtual DOM
  • Le routeur intégré, peu pratique, est souvent remplacé par un module tiers (UI Router)
  • La migration vers Angular 2 n'est pas évidente

Exemple d'un problème de performance avec Angular Gantt

AureliaJS

  • Supporte les languages de demain (ES2015, ES2016, TypeScript)
  • Zoning pour le dirty checking
  • Architecture par module (Chaque fonctionnalité du framework est optionelle et remplaçable)
  • Choix possible de l'implémentation du binding (Proxy)

ReactJS

  • Du virtual DOM (optimise les changements sur le DOM)
  • Très efficace pour construire des composants graphiques
  • On met du HTML dans le JS (avec JSX)
  • Librairie légère, mais moins complète

Angular2

  • TypeScript (ou ES2015)
  • Orienté Web Components
  • Plus de dirty checking
  • Exploite les streams observables RxJS (Reactive Programming)
  • Injection de dépendance digne d'un environnement backend
  • Moins de concepts différents (mais plus puissant)
  • Disponible en beta

javascript (service)
import {Injectable} from "angular2/core"
import {Http} from 'angular2/http';
import 'rxjs/Rx'

@Injectable()
export class CountryService{
    endpoint_url:String = "https://restcountries.eu/rest/v1/region/";

    constructor(http: Http){
        this.http = http;
    }

    getCountriesByRegion (region:String){
        return this.http.get(this.endpoint_url + region).map(res => res.json());
    }
}
